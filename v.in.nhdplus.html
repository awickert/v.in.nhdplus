DESCRIPTION

v.in.nhdplus downloads stream flowlines, drainage catchments, and Watershed Boundary Dataset (WBD) watershed units from NHDPlus and imports them as GRASS vector maps.

Data may be fetched for the current GRASS computational region, or restricted to specific HUC (Hydrologic Unit Code) watersheds via the hucs= option. WBD watershed boundaries at any HUC level (2–12) may be imported separately or in the same run via huc_level=.

By default, NHDPlus version 2 (medium resolution, 1:100,000 scale) is downloaded. NHDPlus HR (high resolution, 1:24,000 scale) may be selected with source=hr. Data are fetched via the pynhd Python library using the USGS Network Linked Data Index (NLDI) and related web services. No authentication is required.

NOTES

Output maps

OptionOutput mapGeometry
type=flowlines <output>_flowlines Lines
type=catchments <output>_catchments Areas (polygons)
type=both (default) both of the above
huc_level=N <output>_hucN Areas (polygons)

Flowlines and catchments are linked by a common identifier: comid in the flowlines table and featureid in the catchments table. Each catchment corresponds to the drainage area of one flowline reach.

Filtering by stream order

For large regions, importing all streams (Strahler order 1 and higher) may return tens of thousands of features. Use min_order= to restrict the import to larger streams:

  • min_order=1 — all streams (default)
  • min_order=3 — streams draining roughly >10 km²
  • min_order=5 — major rivers only

The min_order filter applies to streamorde (Strahler order) and is applied after download. Catchments are not filtered by stream order.

HUC spatial filtering (hucs=)

Instead of the bounding box, flowlines and catchments can be fetched within one or more specific HUC watersheds. Supply HUC codes as a comma-separated list; the level is inferred automatically from code length:

Code lengthLevelApproximate scale
2 digitsHUC2 — Region~large river basins (21 in CONUS)
4 digitsHUC4 — Subregion~222 units
6 digitsHUC6 — Basin~378 units
8 digitsHUC8 — Subbasin~2,250 units; most common for modeling
10 digitsHUC10 — Watershed~22,000 units
12 digitsHUC12 — Subwatershed~160,000 units; finest standard level

All codes supplied must be the same level. The HUC boundaries are retrieved and dissolved to form the spatial filter applied to flowline and catchment queries.

WBD boundary download (huc_level=)

When huc_level= is specified, watershed boundary polygons at that HUC level are imported as a separate map (<output>_hucN). This may be combined with hucs= to download only the specified HUCs, or used alone to import all WBD units intersecting the current region.

NHDPlus v2 vs. NHDPlus HR

NHDPlus v2 (source=v2, default) is derived from the 1:100,000-scale National Hydrography Dataset. It includes rich attribute tables with flow statistics (mean annual flow, velocity) and is the standard for large-scale hydrological analysis. Coverage: contiguous US, Hawaii, Puerto Rico, and US territories.

NHDPlus HR (source=hr) is derived from the 1:24,000-scale NHD and provides greater spatial detail for smaller catchments. It has fewer computed attributes than v2 but is preferred for local-scale analysis. Coverage is expanding but may not cover all regions.

Relationship to v.in.waterdata

v.in.waterdata downloads USGS streamflow gauge time series and upstream basin delineations. v.in.nhdplus downloads the underlying river network geometry and catchments. They are complementary: use v.in.waterdata for discharge observations and v.in.nhdplus for the spatial river framework. Gauge sites can be linked to NHDPlus reaches via their COMID (available in v.in.waterdata output attributes).

EXAMPLES

Import flowlines and catchments within the current region: 
v.in.nhdplus output=nhd
Import only major rivers (Strahler order ≥ 4): 
v.in.nhdplus output=nhd type=flowlines min_order=4
Import flowlines and catchments for specific HUC8 subbasins: 
v.in.nhdplus output=nhd hucs=10190005,10190006
Import everything for a HUC4 subregion, including the HUC8 boundaries: 
v.in.nhdplus output=nhd hucs=1019 huc_level=8
Download WBD HUC12 boundaries within the current region only (no flowlines): 
v.in.nhdplus output=nhd type=flowlines huc_level=12 min_order=99
# (min_order=99 skips flowlines effectively; or set type= to something unused)
Download HUC8 boundaries and flowlines for a HUC2 region: 
v.in.nhdplus output=nhd hucs=10 huc_level=8 min_order=4
Import NHDPlus HR for a small watershed: 
v.in.nhdplus output=nhd_hr source=hr hucs=1019000501
Display stream network coloured by order: 
v.in.nhdplus output=nhd min_order=2
d.vect nhd_flowlines color=blue width_column=streamorde
Join catchments to gauge basin data from v.in.waterdata: 
v.db.join map=nhd_catchments column=featureid \
    otable=gauges_basins ocolumn=comid
